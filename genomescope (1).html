Background reading
What are structural variations?
Before diving into the tool, here's the biology behind it — explained the way we'd explain it to a friend, not a textbook.
The 4 types we looked at
Ways DNA structure can break
Each one creates a different visual signature in the contact map — that's how the algorithm tells them apart.
✂
Deletion
A stretch of DNA simply gets removed. The two regions that used to sit far apart end up next to each other, so they start touching — and that shows up as a bright unexpected block in the contact map.

Think of cutting a few pages out of a book. The page before and the page after now face each other even though they were never meant to.
LeukemiaDiGeorge syndromeCri-du-chat
⧉
Duplication
A region gets copied and pasted. The extra copy means those genomic positions show up in the contact map more often than they should — like a page printed twice making that section feel thicker.

In cancer, duplications often hit oncogenes — genes that, when overactive, push cells to divide uncontrollably.
HER2+ breast cancerCharcot-Marie-ToothAutism
⟷
Inversion
A segment gets cut out, flipped backward, and re-inserted. The contacts it makes now appear in mirrored positions, forming a cross or butterfly pattern in the heatmap.

It's like reversing a sentence — the letters are all still there, but the meaning is completely changed.
Haemophilia AInfertilityProstate cancer
⇄
Translocation
A piece breaks off from one chromosome and attaches to a completely different one. In the contact map, you see a strong signal between two chromosomes that should have nothing to do with each other.

The BCR-ABL translocation (chr9 swapping a piece with chr22) is one of the most studied mutations in medicine — it directly causes CML leukemia.
CML (Philadelphia chr)Burkitt lymphomaAML
Our pipeline
Step by step — what the code actually does
Six stages from raw sequencing reads to a final list of SV candidates.
01

Start with raw Hi-C sequencing reads

The Hi-C experiment physically cross-links DNA that's close together in 3D space, then sequences those pairs. You get millions of read pairs — the raw material for everything that follows.

Illumina Hi-Cin situ Hi-C
02

Map reads back to the reference genome

Each read pair has to be mapped to a position in the human genome (GRCh38). Reads that don't map cleanly or are duplicates get removed — this is just standard quality control.

Bowtie2HiCUPJuicer
03

Build the contact matrix

Count how often each pair of genomic regions was found touching. Store that as a grid. A bright cell at position (row 50, col 300) means regions 50 and 300 contacted each other a lot.

coolerHiC-Pro
04

Normalize to remove technical bias

Some regions just have more DNA, so they naturally appear brighter. ICE normalization adjusts for this so we're comparing apples to apples, not apples to oranges.

ICE normalizationKR balancing
05

Find the anomalies

Contacts naturally decay with distance — nearby regions touch more. Anything that's much brighter than it should be given its distance from the diagonal is a candidate SV. We test this statistically and with an Isolation Forest model.

Z-score testIsolation ForestSaliency map
06

Label each anomaly and write the report

We look at the shape of each anomalous block to figure out what type of SV it is — a deletion looks different from a translocation. Then we output coordinates, type, and a confidence score.

Random Forest classifierPattern matching

Who could actually use something like this?
We kept asking ourselves — if this pipeline worked well, who would benefit? Here's what we came up with.
🧬

Cancer research and diagnosis

Some chromosomal rearrangements happen very early in cancer development — before symptoms appear. If a lab could run Hi-C on a biopsy and flag these rearrangements automatically, it could change how early cancers get caught.

BCR-ABL translocation is the direct cause of CML — our tool flags it
HER2 duplication tells oncologists which targeted therapy to use
Tumor suppressor deletions explain why a cancer became aggressive
Confidence scores help prioritize which findings need follow-up
💊

Choosing the right treatment

Two patients with "the same cancer" can have completely different SVs underneath. What works for one person might do nothing for the other. A tool that maps the actual genomic damage helps doctors pick treatments that make sense for that specific patient.

Match a patient's SV profile to drugs known to target that mutation
Some SVs predict resistance to standard chemotherapy
Families with inherited SVs can get proper genetic counseling
Monitoring changes in SVs over time tracks treatment response
🌾

Crop science (especially relevant in India)

Structural variations in rice and wheat genes directly affect how much grain a plant produces, whether it survives drought, and how well it resists disease. Agricultural researchers are already studying this — our pipeline could help screen crop varieties faster.

Find SVs in OsDREB genes that affect drought tolerance in rice
Screen wheat varieties for yield-boosting duplications
Relevant to ICAR, IARI, and state agricultural universities
No human ethics approval needed — you can start immediately
🔬

Undiagnosed rare diseases

Many families spend years going from doctor to doctor without an answer. A good chunk of those undiagnosed cases involve SVs that standard whole-exome sequencing simply doesn't pick up. Hi-C-based detection could find them.

SVs in regulatory regions break genes without touching their coding sequence
Children with developmental delays often have hidden SVs
Could feed into newborn screening programs eventually
Helps researchers build larger SV databases for rare conditions

Evaluation
How we measure accuracy
Precision
Of all SVs we called, how many were real? Minimizes false alarms.
Recall
Of all real SVs, how many did we find? Minimizes missed detections.
F1 Score
Harmonic mean of precision and recall — the single most important number.
Breakpoint accuracy
How close our predicted breakpoint is to the real one (in base pairs).
Sample Python detection code
# 1. Load and normalize the contact matrix
import numpy as np
from scipy import stats
from sklearn.ensemble import IsolationForest

matrix = np.loadtxt('hic_matrix.csv', delimiter=',')

# 2. Compute observed / expected (flatten distance decay)
def oe_normalize(m):
    oe = np.zeros_like(m, dtype=float)
    for d in range(m.shape[0]):
        diag = np.diag(m, d)
        if diag.mean() > 0:
            oe += np.diag(diag / diag.mean(), d)
    return oe

oe_matrix = oe_normalize(matrix)

# 3. Anomaly detection — Isolation Forest on local windows
features = []
coords = []
W = 5  # window size
for i in range(W, oe_matrix.shape[0]-W):
    for j in range(i+5, oe_matrix.shape[1]-W):
        window = oe_matrix[i-W:i+W, j-W:j+W]
        features.append([window.mean(), window.max(),
                         window.std(), oe_matrix[i,j]])
        coords.append((i, j))

clf = IsolationForest(contamination=0.02).fit(features)
scores = clf.decision_function(features)

# 4. Call SVs from anomalous windows
sv_calls = [(coords[k], scores[k]) for k in range(len(scores))
            if scores[k] < -0.1]
print(f"Detected {len(sv_calls)} candidate SV regions")
